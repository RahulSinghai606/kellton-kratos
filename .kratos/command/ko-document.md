Generate comprehensive documentation for the specified component:

1. **Overview** - Purpose and scope of the component
2. **Architecture** - How it fits in the system
3. **API Reference**:
   - Endpoints/Functions/Classes
   - Parameters and return types
   - Examples
4. **Usage Guide**:
   - Getting started
   - Common use cases
   - Code examples
5. **Configuration** - Available options and defaults
6. **Troubleshooting** - Common issues and solutions

Output formats:
- Markdown for repo docs
- JSDoc/TSDoc for inline docs
- OpenAPI for API specs

Use Context7 for framework-specific documentation patterns.
