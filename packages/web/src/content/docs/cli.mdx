The Kellton Kratos CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
kratos
```

But it also accepts commands as documented on this page. This allows you to interact with Kellton Kratos programmatically.

```bash
kratos run "Explain how closures work in JavaScript"
```

---

### tui

Start the Kellton Kratos terminal user interface.

```bash
kratos [project]
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--prompt`   |       | Prompt to use                              |
| `--model`    | `-m`  | Model to use in the form of provider/model |
| `--agent`    |       | Agent to use                               |
| `--port`     |       | Port to listen on                          |
| `--hostname` |       | Hostname to listen on                      |

## Commands

The Kellton Kratos CLI also has the following commands.

---

### agent

Manage agents for Kellton Kratos.

```bash
kratos agent [command]
```

---

### attach

Attach a terminal to an already running Kellton Kratos backend server started via `serve` or `web` commands.

```bash
kratos attach [url]
```

This allows using the TUI with a remote Kellton Kratos backend. For example:

```bash
# Start the backend server for web/mobile access
kratos web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
kratos attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
kratos agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
kratos agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
kratos auth [command]
```

---

#### login

Kellton Kratos is powered by the provider list at [Models.dev](https://models.dev), so you can use `kratos auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/kratos/auth.json`.

```bash
kratos auth login
```

When Kellton Kratos starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
kratos auth list
```

Or the short version.

```bash
kratos auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
kratos auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
kratos github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
kratos github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
kratos github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
kratos mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
kratos mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
kratos mcp list
```

Or use the short version.

```bash
kratos mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
kratos mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
kratos mcp auth list
```

Or use the short version.

```bash
kratos mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
kratos mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
kratos mcp debug <name>
```

### models

List all available models from configured providers.

```bash
kratos models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
kratos models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Kellton Kratos.

```bash
kratos models --refresh
```

### run

Run kratos in non-interactive mode by passing a prompt directly.

```bash
kratos run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "kratos run"
kratos run Explain the use of context in Go
```

You can also attach to a running `kratos serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
kratos serve

# In another terminal, run commands that attach to it
kratos run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                        |
| ------------ | ----- | ------------------------------------------------------------------ |
| `--command`  |       | The command to run, use message for args                           |
| `--continue` | `-c`  | Continue the last session                                          |
| `--session`  | `-s`  | Session ID to continue                                             |
| `--share`    |       | Share the session                                                  |
| `--model`    | `-m`  | Model to use in the form of provider/model                         |
| `--agent`    |       | Agent to use                                                       |
| `--file`     | `-f`  | File(s) to attach to message                                       |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)              |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided) |
| `--attach`   |       | Attach to a running kratos server (e.g., http://localhost:4096)  |
| `--port`     |       | Port for the local server (defaults to random port)                |

### serve

Start a headless Kellton Kratos server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
kratos serve
```

This starts an HTTP server that provides API access to kratos functionality without the TUI interface.

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### session

Manage Kellton Kratos sessions.

```bash
kratos session [command]
```

---

#### list

List all Kellton Kratos sessions.

```bash
kratos session list
```

##### Flags

| Flag          | Short | Description                          |
| ------------- | ----- | ------------------------------------ |
| `--max-count` | `-n`  | Limit to N most recent sessions      |
| `--format`    |       | Output format: table or json (table) |

---

### stats

Show token usage and cost statistics for your Kellton Kratos sessions.

```bash
kratos stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
kratos export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or Kellton Kratos share URL.

```bash
kratos import <file>
```

You can import from a local file or an Kellton Kratos share URL.

```bash
kratos import session.json
kratos import https://opncd.ai/s/abc123
```

This starts an HTTP server and opens a web browser to access Kellton Kratos through a web interface.

| `--hostname` | Hostname to listen on                      |

### acp

```bash
kratos acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall Kellton Kratos and remove all related files.

```bash
kratos uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates kratos to the latest version or a specific version.

```bash
kratos upgrade [target]
```

To upgrade to the latest version.

```bash
kratos upgrade
```

To upgrade to a specific version.

```bash
kratos upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

Kellton Kratos can be configured using environment variables.

| Variable                              | Type    | Description                              |
| ------------------------------------- | ------- | ---------------------------------------- |
| `KRATOS_AUTO_SHARE`                 | boolean | Automatically share sessions             |
| `KRATOS_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows   |
| `KRATOS_CONFIG`                     | string  | Path to config file                      |
| `KRATOS_CONFIG_DIR`                 | string  | Path to config directory                 |
| `KRATOS_CONFIG_CONTENT`             | string  | Inline json config content               |
| `KRATOS_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks          |
| `KRATOS_DISABLE_PRUNE`              | boolean | Disable pruning of old data              |
| `KRATOS_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates |
| `KRATOS_PERMISSION`                 | string  | Inlined json permissions config          |
| `KRATOS_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                  |
| `KRATOS_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads   |
| `KRATOS_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models               |
| `KRATOS_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction     |
| `KRATOS_CLIENT`                     | string  | Client identifier (defaults to `cli`)    |
| `KRATOS_ENABLE_EXA`                 | boolean | Enable Exa web search tools              |

---

### Experimental

These environment variables enable experimental features that may change or be removed.

| Variable                                        | Type    | Description                             |
| ----------------------------------------------- | ------- | --------------------------------------- |
| `KRATOS_EXPERIMENTAL`                         | boolean | Enable all experimental features        |
| `KRATOS_EXPERIMENTAL_ICON_DISCOVERY`          | boolean | Enable icon discovery                   |
| `KRATOS_EXPERIMENTAL_DISABLE_COPY_ON_SELECT`  | boolean | Disable copy on select in TUI           |
| `KRATOS_EXPERIMENTAL_BASH_MAX_OUTPUT_LENGTH`  | number  | Max output length for bash commands     |
| `KRATOS_EXPERIMENTAL_BASH_DEFAULT_TIMEOUT_MS` | number  | Default timeout for bash commands in ms |
| `KRATOS_EXPERIMENTAL_OUTPUT_TOKEN_MAX`        | number  | Max output tokens for LLM responses     |
| `KRATOS_EXPERIMENTAL_FILEWATCHER`             | boolean | Enable file watcher for entire dir      |
| `KRATOS_EXPERIMENTAL_OXFMT`                   | boolean | Enable oxfmt formatter                  |
| `KRATOS_EXPERIMENTAL_LSP_TOOL`                | boolean | Enable experimental LSP tool            |
